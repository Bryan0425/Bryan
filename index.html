Ubuntu VPS上搭建Shadowsocks IPV6代理

Posted on 2018-05-08

老板最近抱怨上不去Google、YouTube啊（她是MAC用户），然后偶然间听我说我用着北京实验室的代理畅通无阻，就给我下达了立即购买部署的任务。情急之下，我请教了去年在北京学习时认识的最牛逼的师兄————Kevinzjy，我亲爱的无所不知无所不能的酱哥，原谅我就是这样一个酱吹。然后在他的帮助下，搞定了这个小任务。

以下为基本操作，大多依照酱哥的protocol，稍有变化改动。原文链接：https://kevinzjy.github.io/2017/11/05/Ubuntu-shadowsocks/

选用DigitalOcean上的VPS

至于原因嘛，酱哥聊天先发了这个…

Step1: 建立Server

在DigitalOcean上面新建一个Droplet，我选择的是美国纽约

Step2: 登录VPS进行配置

默认下将IP地址root密码发送至邮箱
修改Root密码

ssh root@vps_ip # 登录VPS
sudo passwd root #修改ROOT密码

Enter new UNIX password: # 输入新密码

passwd: password updated successfully

修改时区

dpkg-reconfigure tzdata #修改时区，选择Asia/Shanghai

更新APT源

apt update # 更新apt源
apt upgrade # 升级

推荐新建一个具有Root权限的用户，方便VPS的管理和操作

sudo adduser panb # 新建用户
# 在 /etc/sudoers 中添加root权限
# User privilege specification
panb ALL=(ALL) ALL

下面的操作可以退出后使用新建的账户进行操作

Step3: 安装Shadowsocks服务

DigitalOcean上的VPS最大的作用就是可以同时连接ipv4和ipv6服务，在搭建Shadowsocks服务后，可以在校园网下用IPV6代理实现直接访问IPV4网址，不需要登录网关，同时避开了流量计费系统。

sudo apt install python-pip
sudo pip install --upgrade pip
sudo pip install setuptools
sudo pip install shadowsocks

特别注意的是，有的服务器自带setuptools ，卸载之后重装就好了。

编辑vim /etc/shadowsocks.json，添加如下内容

{
    "server": "::", # 监听IPV6地址
    "server_port": 443, # 使用443端口，更加“安全”
    "local_port": 1080,
    "password": "PASSWORD", # 密码
    "timeout": 600,
    "method": "aes-256-cfb"
}

注意：我的版本里运行时需删掉注释，否则不work。

现在，可以手动开启shadowsocks服务了

ssserver -c /etc/shadowsocks.json -d start

以后的重启就是-d restart

Step4: 配置shadowsocks服务开机自动启动

据酱哥说一直不成功哈哈哈~

创建脚本 /etc/init.d/shadowsocks

#!/bin/sh
### BEGIN INIT INFO
# Provides:          shadowsocks
# Required-Start:    $remote_fs $syslog
# Required-Stop:     $remote_fs $syslog
# Default-Start:     2 3 4 5
# Default-Stop:      0 1 6
# Short-Description: start shadowsocks 
# Description:       start shadowsocks
### END INIT INFO
ssstart(){
    /usr/local/bin/ssserver -c /etc/shadowsocks.json -d start
}
ssstop(){
    /usr/local/bin/ssserver-d stop
}
ssreload(){
    /usr/local/bin/ssserver -c /etc/shadowsocks.json -d restart
}
case "$1" in
start)
    ssstart
    ;;
stop)
    ssstop
    ;;
reload)
    ssreload
    ;;
*)
    echo "Usage: $0 {start|reload|stop}"
    exit 1
    ;;
esac

增加文件的执行权限，并加入rc.d中，实现开机启动

sudo chmod +x /etc/init.d/shadowsocks
sudo update-rc.d shadowsocks defaults

手动开启shadowsocks服务

sudo service shadowsocks start

Step5: shadowsocks自动重连

创建autostart.sh

#!/bin/bash
pids="$($_CMD pgrep ssserver)"
if [ ! $pids ]; then
        /usr/local/bin/ssserver -c /etc/shadowsocks.json -d start >> /root/ss.log
        newid="$($_CMD pgrep ssserver)"
        echo "Shadowsocks restarted at pid: $newid" >> /root/ss.log
fi

在crontab配置中，配置每天凌晨重启，以及每分钟检测断线重连

crontab -e 
# 选择[3]: vim.basic
# 最后加入
0 4 * * * ssserver -c /etc/shadowsocks.json -d restart >> /root/ss.log
*/1 * * * * /root/autostart.sh
重启Crontab，使定时任务生效

service cron reload
